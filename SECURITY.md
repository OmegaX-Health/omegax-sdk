# Security Policy

## Supported versions

Security fixes are prioritized for the latest published release line.

## Reporting a vulnerability

Please do **not** open public issues for security reports.

Use one of these channels:

1. Open a private GitHub Security Advisory in this repository, or
2. Contact the maintainers through your existing private support channel.

Please include:

- Affected package version
- Impact summary
- Reproduction steps or proof of concept
- Suggested remediation (if available)

We will acknowledge valid reports as quickly as possible and coordinate a fix and disclosure timeline.
